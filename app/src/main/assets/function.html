3.1 用户管理

3.1.1 功能设计

用户管理主要是管理、各级、各类用户的基本信息和使用权限等。主要实现以下功能：
1. 用户需要分多种类型，如：系统管理员、社区用户、家庭用户等；
2. 用户注册管理、用户登录管理；
3. 用户基本信息管理，如：姓名、电话号码、email、住址等；
4. 用户状态管理，非活动状态用户禁止登陆系统；
5. 用户密码机制；
6. 用户意见箱，方便用户系统内与管理员交流。

3.2 权限管理

3.2.1 功能设计

用户权限主要是实现不同用户在系统内访问资源的可控性，同时也是保护用户的使用安全。主要实现以下功能：
1. 登陆用户仅可以查看自己相关资源，如家庭用户尽可以查看自己家庭所关联的监控节点信息；
2. 视频监控信息仅直接关联用户可查看，其他任何用户无权查看，包括超级管理员，以保护个人隐私；

3.3.1 功能设计

1. 实现监控节点信息管理，系统内可以查询每个监控节点的坐标信息（用于地图集成）、基本信息（厂家、品牌、规格、关键参数等）；
2. 集成地图或社区平面图（地图优先），监控节点设备可在地图只管展示，并通过点击的方式选取；
3. 监控节点当前状态显示，如摄像头可切换至摄像头当前实时影像、温度感应器可展示当前温度走势等；
4. 可关联展示当前节点的历史告警记录；
5. 通过历史数据分析，展示事故最多的监控点以便于用户针对性的防范。

3.4 资源设备管理

资源设备主要管理对象包括视频采集设备、温度感应设备、红外感应设备、蓝牙设备以及系统部署的服务器等。主要管理内容包括设备基本信息、设备状态信息、设备巡检记录、设备维护记录等。主要功能如下：

3.4.1 功能设计

1. 设备基本信息管理，包括设备编码、设备厂家、设备类型、IP地址、当前状态等；
2. 摄像头管理，摄像头厂家、分辨率、视频流配置、摄像头远程查看等；
3. 视频存储管理，摄像头录制的影像资料存储；
4. 设备巡检记录，记录设备的巡检结果及相关信息；
5. 设备维护记录，记录设备每次维护的历史记录信息。

3.5 录像配置管理

针对每个摄像头，系统均可以设置相应的录制配置和录制计划，以满足不同监控节点的多种需求。主要功能入下：
3.5.1 功能设计
1. 录像参数设置，可设置录制视频流名称、自动编码规则、所属摄像头、视频是否存储及保存天数、视频保存格式、视频上传服务器等信息；
2. 录像计划配置，包括所属摄像头、计划录制日期、开始录制时间、结束录制时间（细化到分钟）等信息；

3.6 录像存储管理

录像存储主要管理摄像头录制的视频文件上传至服务器。并针对不同权限供用户查询播放。主要包括以下功能：
3.6.1 功能设计
1. 视频检索查询，可根据用户、服务器IP、视频名称、视频格式、时间范围（以开始录制时间为基准）；
2. 视频信息管理，包括所属服务器名称、服务器ID、视频格式、文件名称、所属摄像头、上传时间等；
3. 视频在线播放。

3.7 告警管理

告警管理是系统功能中的关键点之一，安全管理的目标就是在预警的情况下提前预知危险的存在，通过告警的方式通知用户，避免灾害的发生。
3.7.1 功能设计
1. 告警代码定义，首先要梳理系统告警规范，将告警信息标准化，告警代码信息包括告警编号、告警类型、告警设备、告警内容、安全措施提示等内容，告警代码需要与硬件设备集成，对应相应的硬件设备告警信息；
2. 告警日志管理，主要用于记录各种设备（包括服务器、摄像头、检测设备等）的告警记录，主要内容包括告警设备编码、设备名称、设备当前状态、告警时间、告警代码等内容；
3. 告警传送管理，告警传送也是告警管理给用户最直观的展示，当硬件设备满足告警条件，产生告警后，系统可以自动判断告警代码及内容，并将告警信息及处理措施以短信、彩信、email等方式传递给用户；
4. 摄像头告警信息传送给用户后，用户可以通过链接的形式进入web端或手机APP端查看当前摄像头的实时影像信息；
5. 温感等检测设备告警信息传送给用户后，用户可以通过链接的形式进入WEB端或手机APP端查看当前节点设备的数据走势曲线，并可以选择切换到临近的摄像头查看实时影像；
6. 系统存储历史告警记录信息，通过大数据分析给管理者提供各节点的安全隐患发生率或告警频率等信息，以便采取针对性的措施，避免灾害发生。
